# 📁 docs/ — Documentação

Esta pasta contém toda a **documentação do projeto**: explicações, guias, diagramas e referências que ajudam qualquer pessoa a entender como o projeto funciona.

---

## O que é essa pasta?

`docs` vem da palavra inglesa **documentation**.  
Enquanto o código explica *como* algo é feito, a documentação explica *o que* é feito e *por quê*.

É especialmente útil quando:
- Alguém novo entra no projeto
- Você retorna a um código que não vê há semanas
- Precisa explicar o projeto para um professor ou colega

---

## Tipos de arquivo que pertencem aqui

| Extensão | O que é |
|---|---|
| `.md` | Markdown — texto formatado, fácil de escrever e ler no GitHub |
| `.pdf` | Documentos mais formais, relatórios, especificações |
| `.png` / `.jpg` | Diagramas, fluxogramas, imagens de apoio |
| `.drawio` | Diagramas editáveis (ferramenta draw.io) |

---

## Exemplos de arquivos que ficam aqui

```
docs/
├── estrutura-do-projeto.md   ← explica as pastas e a organização geral
├── como-compilar.md          ← passo a passo para compilar o projeto
├── fluxograma.png            ← imagem de como o programa funciona
└── requisitos.md             ← o que o programa deve fazer
```

---

## Por que usar essa pasta?

Sem documentação, um projeto pode se tornar impossível de entender — mesmo para quem o criou.  
Centralizar tudo em `docs/` mantém a raiz do projeto limpa e diz claramente onde encontrar qualquer explicação sobre o sistema.

Projetos bem documentados são valorizados em processos seletivos, trabalhos acadêmicos e no mercado de trabalho.

---

## Como escrever um arquivo `.md`?

Markdown é uma linguagem simples de formatação. Alguns exemplos:

```
# Título grande
## Título médio
**negrito**
- item de lista
[texto do link](https://url.com)
```

O GitHub renderiza automaticamente qualquer arquivo `.md` como uma página formatada.
